# Grok API Travel Info Setup Guide

## 🚀 Feature Overview

This feature adds AI-powered travel information for destinations using Grok API. When users search for hotels, they can switch to the "About [City]" tab to get comprehensive travel guides dynamically generated by Grok.

## 📋 What You Get

- **Dynamic Travel Guides**: AI-generated content for any destination
- **Tab Interface**: Clean separation between hotel results and destination info
- **Structured Information**:
  - 🏛️ Tourist Places
  - 🍜 Famous Food
  - 🌤️ Best Time to Visit
  - 🎯 Activities
  - 💡 Travel Tips

## 🔑 Getting Your Grok API Key

### Step 1: Sign up for Grok API Access

1. Visit [x.ai](https://x.ai) or [console.x.ai](https://console.x.ai)
2. Create an account or sign in
3. Navigate to API Keys section
4. Generate a new API key
5. Copy the key (it will look like: `xai-...`)

### Step 2: Add API Key to Backend

Open the backend environment file:

```bash
cd website-copilot/server
nano .env
```

Replace `your_grok_api_key_here` with your actual key:

```env
GROK_API_KEY=xai-your-actual-key-here
```

Save and close the file.

## ✨ How to Use

### 1. Search for Hotels

Enter a query like:
```
3 rooms in Goa, 4 star, cheap, check-in 23 Feb, check-out 26 Feb
```

### 2. View Results

You'll see two tabs:
- **🏨 Hotels (24)** - Your search results
- **📍 About Goa** - Travel information

### 3. Switch to Place Info Tab

Click on "📍 About Goa" to see:
- Loading animation while Grok generates content
- Comprehensive travel guide with structured sections
- All information is dynamically generated (not hardcoded)

## 🎯 How It Works

### Backend Flow

1. Frontend sends place name to `/api/place-info`
2. Backend calls Grok API with structured prompt
3. Grok generates travel guide content
4. Backend returns formatted response

### Frontend Flow

1. User searches for hotels
2. Place name extracted from results (e.g., "Goa")
3. Tab interface shows "Hotels" and "About [Place]"
4. Clicking "About" tab triggers API call (lazy loading)
5. Content displayed with loading state

## 🔧 Customization

### Change Prompt Structure

Edit `website-copilot/server/grokService.js`:

```javascript
const prompt = `You are a travel guide expert. Provide information about ${placeName}.

Add your custom sections here...`;
```

### Adjust Response Length

In `grokService.js`, modify:

```javascript
max_tokens: 1500  // Increase for longer responses
```

### Change Model

```javascript
model: 'grok-beta'  // Use different Grok model if available
```

## 🐛 Troubleshooting

### Error: "GROK_API_KEY not configured"

**Solution**: Add your API key to `.env` file in `website-copilot/server/`

### Error: "Grok API failed: 401"

**Solution**: Your API key is invalid. Get a new one from x.ai console

### Error: "Grok API failed: 429"

**Solution**: Rate limit exceeded. Wait a few minutes or upgrade your plan

### Tab shows "No information available"

**Solution**: 
- Check backend logs for errors
- Verify API key is correct
- Ensure backend server is running

### Loading forever

**Solution**:
- Check network tab in browser DevTools
- Verify backend endpoint is responding
- Check for CORS errors

## 💰 API Costs

Grok API pricing (check x.ai for current rates):
- Pay per token used

## 🚀 Performance Tips

### 1. Cache Responses

Add caching to avoid repeated API calls:

```javascript
const cache = new Map();

async function getPlaceInfo(placeName) {
    if (cache.has(placeName)) {
        return cache.get(placeName);
    }
    
    const result = await fetchFromGrok(placeName);
    cache.set(placeName, result);
    return result;
}
```

### 2. Lazy Loading

The feature already uses lazy loading - API is only called when user clicks the tab.

### 3. Error Handling

Graceful fallback if API fails - shows error message instead of breaking the app.
